Travel API

API Authentication is made using basicAuth (username + password) together with api_key header
Response is always json type on all endpoints
Swagger https://insuretech-rca-online.staging.insuretech.ro/api/v1/swagger-ui/index.html
Test/Staging Env Base URL https://insuretech.staging.insuretech.ro

Version: 1.0.0
Last Update: 27.08.2025 11:50 AM

Description

Travel API consists of the following steps:

obtaining Travel products
filling the minimal information about the trip and check product eligibility
create an order with client/insured details
retrieving offer request bodies. Once the trip information is sent and the desired products are chosen the API responds with all PRE-BUILT offer request bodies that can be directly used to obtain offers from the server. This helps the client to have each request in control and not be blocked by some requests that may take longer to respond to.
send requests with the above PRE-BUILT request bodies and obtain prices for the chosen products
download offer PDF
choose the desired offer and issue the policy
download policy PDF
